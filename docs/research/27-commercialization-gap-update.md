# 27 · 距离商业化交付还有多远(2026-07-23 复盘,接调研 19)

> 调研 19(2026-07-18)在 M0 起点判"业界九项交集缺七、距单桌面商用 2–3 年"。
> 之后 R1–R4 主体 + 动画格式 + sv-lsp + 增量布局逐批落地(远端至 PR #46)。
> 本篇按同一分档口径复盘**当前**距离,缺口一律带代码/文档证据。

## 0. 一句话判决(更新)

**造的部分基本造完了,验的部分几乎还没开始。** 调研 19 的 2–3 年估算里最贵、
先例证明最耗时的整段——**自绘文本栈(输入/IME/换行/shaping)+ flex 布局 +
无障碍 + 弹层**——现在都落地且有测试。剩下的不再是"能不能做出来"的能力未知,
而是三类**收敛与验证**:① 一件真正的架构大件(双前端内核合并,API 冻结前置);
② 一个卡发布的**决策**(改名 ADR-10,非工程);③ 一片几乎空白的
**真机/真素材/真用户验证**盲区——先例里那条"以年计"的长尾。

## 2. 分档现状

- **档 A(内部工具可用):达成。** R1 输入 + R2 视口布局全落地,settings-sfc 验收通过。
- **档 B(单桌面可商用)特性面:基本达成。** R3(文本栈/无障碍/弹层)+ R4 工程侧
  (去 panic、clippy 阻塞门禁、MSRV 道、cargo-deny、发布体检、打包钩子)+ ADR-6 落地。
  DESIGN.md §5 已自标"档 B 达成"。
- **档 C(四平台/鸿蒙):零起步。** R5 未动,窄窗口 trait 未从 winit 绑定抽出——接入点不存在。

## 3. 真正剩下的(诚实清单)

把"档 B 达成 on paper"与"敢让第三方商用交付"分开,挡在中间的是:

**3.1 API 稳定性(唯一剩的架构大件)**
双前端内核合并(`view!` 与 `.sv` 目前 ADR-2 共存)是 API 冻结最后前置;
ADR-6 已落地,`on:` 语法收敛未做。**没冻结 = semver 承诺无法启动**,
而 semver-checks 又要已发布基线(先有鸡先有蛋)。DESIGN.md §5 R4:
"API 冻结前置只剩双前端内核合并"。

**3.2 一个卡发布的决策**
改名 ADR-10 待裁决 + 商标粗查 + org 查重(DESIGN.md ADR-10)。纯决策/法务。

**3.3 真机/真素材验证盲区(先例里那条长尾)** —— 最该强调的诚实点:
- IME 只离屏验过,**三平台真机手测未做**;
- 无障碍有金样测试,**NVDA/VoiceOver/Orca 真机朗读冒烟未做**(a11y.rs 待办);
- **PAG 从未在真实 `.pag` 上验过**(`sv-pag` "头号缺口";固件是手工构造 + 真 WebP 块);
- 惯性/触摸滚动要真机;签名/公证是组织级账号事务(工程侧只到钩子)。

**3.4 生态与打磨**
组件库 sv-arco(调研 26 有可行性、零代码)、文档站、真实示例应用、
崩溃报告/日志框架(调研 19 列的交付面空白仍在)。

## 4. 结论:"距离"两种读法

- **能力就绪度**:极近。八/九交集达标,档 B 特性齐;搭出真桌面应用缺的是
  双前端合并(收敛)+ 改名(决策)+ 发版——**以人周计,不是年**。推进速度
  远超所有对标先例(Slint 3 年、iced 6 年、egui 2 年)。
- **交付信心(生产硬化)**:仍是长尾入口。真机 IME、屏幕阅读器、真 PAG 素材、
  真实应用、签名分发——先例证明以年计,svelte-rs 几乎还没进入。
  **486 测试全绿能证明"造好了",证明不了"真实世界里扛得住"。**

## 5. 从现在到 crates.io 首发的最短关键路径(按依赖序)

1. **`on:` 事件语法收敛**(小 breaking,与冻结同批出清);
2. **双前端内核合并**(独立大件,API 冻结前置);
3. **去 panic 收尾**(sv-shell 非测试 expect 复审)+ **fuzz 扩面**(解析器类);
4. **改名裁决 ADR-10**(决策;之后 publish 流水线从"体检"转"真推");
5. **六 crate 依赖序首发** + `cargo-semver-checks` 接基线;
6. **真机验证矩阵**(IME ×3 平台 + 屏幕阅读器冒烟 + 真 PAG 素材)——档 B 信心的关键;
7. 组件库 sv-arco 起步 + 文档站 → 生态；鸿蒙 R5 → 档 C。
