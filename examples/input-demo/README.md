# input-demo:文本输入 / IME / 剪贴板手测清单

调研 21 §3 的裁决:IME 逻辑 ~90% 已离屏自动化(见 sv-ui `input` 模块与
todo-sfc/input-demo 行为测试),但**真输入法全流程 OS 会话无法合成**——iced/egui
的 IME 修复全部来自真机反馈。发版前按下表逐项勾选。

运行:`cargo run -p input-demo`

## 手测矩阵(三平台 × 各一款输入法)

| 平台 | 输入法 | 状态 |
|---|---|---|
| Windows | 微软拼音 | ☐ |
| macOS | 系统拼音 | ☐ |
| Linux X11 | fcitx5 或 ibus | ☐ |
| Linux Wayland | fcitx5(合成器 text-input 协议) | ☐ |

## 每平台 5 项勾选

1. **候选窗贴光标**:点击输入框键入拼音,候选窗应出现在光标正下方;
   光标移动/继续键入时跟随(`set_ime_cursor_area` 链路)。
2. **预编辑内联**:组合中拼音以下划线内联显示在框内(over-the-spot),
   `bind:value` 的实时值**不含**组合中的半成品文本。
3. **选字上屏**:空格/数字选字后文本入值,`实时值` 行同步;连续组合无残留。
4. **Esc 取消组合**:预编辑消失、值不变;再次键入正常。
5. **跨应用剪贴板**:在别处复制中文 → Ctrl+V 粘贴进来;Ctrl+A/C 全选复制 →
   粘贴到系统记事本;(X11 注意:无剪贴板管理器时内容随进程退出消失,业界通病)。

附加(有输入框焦点时):按键重复(长按 Backspace)、Shift+方向键扩选、
Home/End、Tab 移出输入框后 IME 会话应关闭(任务栏输入法状态复位)。

## 已知边界(v0,调研 21 §5)

- 拖拽选择、双击选词、undo/redo、多行 TextArea:档 B(步骤 6 + M2 Parley)。
- 溢出滚移是绘制层每帧无状态计算,点击定位在长文本溢出时略有偏差。
- 外部响应式写入恰逢组合中:裁决为"清预编辑",等真实反馈再调。
