# sv-vap

VAP(腾讯 Video Animation Player)素材解析与 RGB+alpha 合成。**纯 Rust,零依赖。**

## 起因:一批"买来是 json + mp4"的礼物素材

线上买的直播礼物特效,交付物是 `xxx.json` + `xxx.mp4` + 预览 `png`,
而卖家说"用 PAG 播放"。这让人以为缺文件了 —— **并没有**。

那是 **VAP**,不是 PAG。两者都出自腾讯、都用在直播礼物上、业内口头常混着叫,
但**文件格式毫无关系**:

| | PAG(见 `sv-pag`) | VAP(本 crate) |
|---|---|---|
| 载体 | 单个 `.pag` 二进制(magic `PAG`) | **MP4 + 一段 JSON 配置** |
| 内容 | 矢量图层树 / 位图序列 / 视频序列 | H.264 视频 |
| alpha | 格式自带 | **H.264 没有 alpha**,于是把 alpha 当灰度画面并排塞进同一帧 |

所以 "json + mp4" 就是 VAP 的**正常交付形态**。而且配置在 MP4 里**也内嵌了一份**
(一个类型为 `vapc` 的自定义顶层 box),旁车 JSON 只是副本 —— 只发 mp4 也能播。

## 一帧长什么样(实测)

```text
        <-------- videoW = 1136(对齐到 16 的倍数)-------->
      +------------------+----+-----------+--+
      |                  |填充|           |填|   videoH
      |   RGB 750x1624   |    | alpha     |充|   = 1632
      |   (rgbFrame)     |    | 375x812   |  |
      |                  |    | (aFrame)  |  |
      |                  +----+-----------+--+
      |                  |      填充(黑)     |
      +------------------+-------------------+
```

- alpha 区是**灰度**(R=G=B),取任一通道即可;
- alpha 是**半分辨率**(375×812 对 750×1624),要放大采样 —— 这是最容易写错的一处;
- 是**直通 alpha,不是预乘**;
- `videoW=1136` 大于内容右边界 1129,因为 H.264 要对齐宏块。

## 本 crate 做什么 / 不做什么

**做**:读 `vapc` 配置(从 MP4 box 或旁车 JSON)、校验几何、把一帧解码后的 RGB
合成成 RGBA(直通或预乘两种口径)。

**不做:H.264 解码。** 那要引一个视频解码器,是独立的一次重裁决,且与平台强相关。
与 `sv-pag` 把 WebP 解码挡在外面是同一条纪律:**这层只有一个职责。**

## 真实素材验证(2026-07-22)

`sv-pag` 的头号缺口是"从未在真实文件上验证过"。**这条线没有重蹈覆辙。**

10 个线上礼物素材(750×1624 显示 / 1136×1632 视频 / 101–240 帧 / 24 或 30fps)全部:

- 从 MP4 的 `vapc` box 读出配置 ✅
- 与旁车 `.json` 的解析结果**逐字段一致** ✅
- 用 ffmpeg 解一帧后合成成功,预乘口径全图满足 `r,g,b <= a` ✅
- 合成结果与一份**独立写的 Python 参考实现逐字节相同** ✅

复现:`cargo run -p sv-vap --example verify_real_asset -- <某个.mp4>`
(素材是商用资源,不入库,所以这是 example 不是测试。)

### 真实素材当场纠正了我一个错误假设

验证脚本原本断言"alpha 应当同时有透明区与不透明区,否则多半是 `aFrame` 取错了"。
10 个素材一跑,当场被打脸两次:

- **流星背景图** —— alpha **全程恒 255**,它就是个全屏不透明背景;
- **糖果扫把** —— 细长物体,画面绝大部分是空的,不透明像素不足 1%。

两个都完全合法。**真实素材的形态跨度比直觉大得多**,任何对"内容长什么样"的
断言都会变成误报。现在只报告分布,不断言。

## 已知缺口

1. **VAPX(融合动画)不支持**。`isVapx=1` 时素材带运行期动态元素(头像、昵称),
   本 crate 解析成功并把标志透出,但**不处理那些元素**。手上 10 个素材全是 0。
2. **没有解码器**,见上。
3. **只见过一种布局**(alpha 在右、半分辨率)。VAP 允许 alpha 在下方、
   或等分辨率 —— 代码按配置里的矩形走、不假设方位,`one_to_one_alpha_needs_no_scaling`
   测了等分辨率那条路,但**没有真实的"alpha 在下方"素材可验**。
4. **音频没管**。这些 mp4 带 AAC 音轨(礼物音效);播放侧要不要出声是上层的事。
5. 手写 JSON 取值器**假设值里不含转义引号**(实测 `codeTag` 是纯 ASCII 标识串)。
